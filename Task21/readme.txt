This document outlines the process of managing users and role assignments within Microsoft Entra ID via the Microsoft Azure Portal. The Entra ID dashboard is accessed to create a new user by providing essential details such as name and username. After successful user creation, the Roles and Administrators section is used to assign an appropriate access role. A basic Reader role is selected to grant limited, read-only permissions to Azure resources. Finally, the configuration is verified to ensure the user account and role assignment are correctly applied. This exercise demonstrates fundamental identity and access management (IAM) operations in Azure.
