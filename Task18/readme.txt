This document outlines the procedure to deploy and configure a basic Virtual Machine using the Microsoft Azure Portal. A Resource Group is created to logically organize associated resources. Within this group, a Virtual Machine is provisioned using a selected image such as Ubuntu Server or Windows Server. The VM size is configured as Standard B1s (or the default sandbox configuration), and authentication is established via password or SSH key. Upon successful deployment, the VM status is verified as Running from the Azure dashboard. Finally, secure remote access is established through CMD using SSH (for Linux) or RDP (for Windows), and screenshots are captured to document successful configuration.
